2.1 Insertion sort: in-place, stable

1.. for j = 2 to A.length
2..	  key = A[j]  
3..   // Insert A[j] into the sorted sequence A[1..j-1]
4..	  i = j-1
5..	  while i > 0 and A[i] > key
6..		  A[i+1] = A[i]
7..		  i = i - 1
8..	  A[i+1] = key
  • Three properties of a loop invariant (similar to mathematical induction) - initialization: true prior to the first iteration of the loop - maintenance: true before the next iteration - termination: gives a useful property that shows the correctness when the loop terminates

Pseudocode conventions

  • after a for loop, the loop counter’s value is the value that first exceeded the for loop bound
  • to, downto ... (by) : increment or decrement its loop counter (default by =1)
  • .. : to indicate a range of values within an array
  • x and y point to the same object after the assignment y = x
  • boolean operators “and” and “or” are short circuiting (left-side first)

2.2 Analyzing algorithms

  • analyzing: measuring computational time of an algorithm
  • assume a generic one processor, random-access machine (RAM) model of computation - RAM model instruction: arithmetic, data movement, control (branch, subroutine) - no concurrent operations in the RAM model
  • input size: depends on the problems → # of items, # of bits, (# vertices, # edges), etc.
  • running time: the number of primitive operations, or "steps", executed - one line in pseudocode takes a constant time - usually concentrate on finding only the worst-case running time (upper bound) - consider only the leading term of a formula w/o a constant ⇒ order/rate of growth

Analyzing insertion sort

  • The test is executed one time more than the loop body
  • Worst case: quadratic function of n; T(n) = an2 + bn + c

2.3 Designing algorithms

2.3.1 The divide-and-conquer approach

  • recursive algorithms typically follow a divide-and-conquer approach ("similar but smaller")
  • MERGE(A, p, q, r)
1.. n1 = q - p + 1
2.. n2 = r - q
3.. Let L[1..n1+1] and R[1..n2+1] be new arrays
4.. for i = 1 to n1
5..   L[i] = A[p+i-1]
6.. for j = 1 to n2
7..   R[j] = A[q+j]
8.. L[n1+1] = inf  // sentinel value
9.. R[n2+1] = inf  // sentinel value
10..i = 1
11..j = 1
12..for k = p to r
13..  if L[i] <= R[j]
14..    A[k] = L[i]
15..    i = i + 1
16..  else A[k] = R[j]
17..    j = j + 1
  • MERGE-SORT(A, p, r)
1.. if p < r
2..   q = └(p+r)/2┛
3..   MERGE-SORT(A, p, q)
4..   MERGE-SORT(A, q+1, r)
5..   MERGE(A, p, q, r)

2.3.2 Analyzing divide-and-conquer algorithms

  • recurrence equation: describes the overall running time on a problem of size n in terms of the running time on smaller inputs - division yields a subproblems, each of which is 1/b the size of the original - D(n): time to divide, C(n): time to combine subproblems solutions
  • for MERGE-SORT: cn lg n + cn ⇒ Θ(n lg n)
