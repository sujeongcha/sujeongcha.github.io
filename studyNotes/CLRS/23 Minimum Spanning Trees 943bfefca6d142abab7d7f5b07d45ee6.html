23.1 Growing a minimum spanning tree

  • greedy strategy; grow a minimum spanning tree one edge at a time - loop invariant: prior to each iteration, A is a subset of some minimum spanning tree - safe edge: adding a safe edge to A maintains the loop invariant
  • Rule for recognizing safe edges

23.2 The algorithm of Kruskal and Prim

Kruskal's algorithm: O(Elg⁡E)=O(Elg⁡V)O(E \lg E)=O(E \lg V)O(ElgE)=O(ElgV)﻿

  • find a safe edge by finding, of all the edges that connect any two trees in the forest, an edge (u,v) of least weight
  • uses a disjoint-set data structure to maintain several disjoint sets of elements

Prim's algorithm: O(Elg⁡V)binary min heapO(E \lg V)_{binary~min~heap}O(ElgV)binary min heap​﻿, O(E+Vlg⁡V)fibonacci heapO(E+V\lg V)_{fibonacci~heap}O(E+VlgV)fibonacci heap​﻿

  • qualifies as greedy since at each step it adds to the tree an edge that contributes the minimum amount possible to the tree’s weight
  • all vertices that are not in the tree reside in a min-priority queue Q based on a key attribute
