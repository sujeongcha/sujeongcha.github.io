11. Hash Tables

Created@Jun 30, 2020 12:12 PM
  • A hash table is an effective data structure for implementing dictionaries

11.1 Direct-address tables

  • assume that no two elements have the same key
  • each slot = a key in the universe U; slot k points to an element in the set with key k
  • SEARCH/INSERT/DELETE takes only O(1)O(1)O(1)﻿ time
  • can store the object in the slot itself, thus saving space

11.2 Hash tables

  • use a hash function h to compute the slot from the key k → Instead of a size of |U|, the array can have size m
  • collision: two keys may hash to the same slot

Collision resolution by chaining

  • chaining: place all the elements that hash to the same slot into the same linked list

Analysis of hashing with chaining

  • load factor α\alphaα﻿ = n/mn/mn/m﻿ (for n elements and m slots)
  • under the assumption of simple uniform hashing - equally likely to hash into any of the m slots - unsuccessful search average-case time: Θ(1+α)\Theta(1+\alpha)Θ(1+α)﻿ - successful search average-case time: Θ(1+α)\Theta(1+\alpha)Θ(1+α)﻿

⇒ α=n/m=O(m)/m=O(1)\alpha = n/m = O(m)/m=O(1)α=n/m=O(m)/m=O(1)﻿ if the #of slots is at least proportional to the # of elements

⇒ Insertion: O(1)O(1)O(1)﻿, delete: O(1)O(1)O(1)﻿ if doubly linked, search: O(1)O(1)O(1)﻿ on average

11.4 Open addressing

  • each table entry contains either an element of the dynamic set or NIL
  • the advantage of open addressing is that it avoids pointers altogether - extra memory freed by not storing pointers provides the hash table with a larger number of slots for the same amount of memory, potentially yielding fewer collisions and faster retrieval
  • Instead of being fixed in the order 0, 1, ..., m-1, the sequence of positions probed depends upon the key being inserted → probe sequence, ⟨h(k,0),h(k,1),...h(k,m−1)⟩\langle h(k,0), h(k,1), ... h(k, m-1) \rangle⟨h(k,0),h(k,1),...h(k,m−1)⟩﻿, is a permutation of 0, 1, .., m-1
  • The algorithm for searching for key k probes the same sequence of slots that the insertion algorithm examined when key k was inserted.
  • use the special value DELETED instead of NIL when deleting
