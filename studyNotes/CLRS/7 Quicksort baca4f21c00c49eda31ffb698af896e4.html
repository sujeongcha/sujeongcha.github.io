7.1 Description of quicksort

  • Applies the divide-and-conquer paradigm - Divide Partition into two → Conquer sort two subarrays recursively → Combine (no work is needed to combine)

Partitioning the array

  • x=A[r]x=A[r]x=A[r]﻿ as a pivot element; partition the array into four regions; Θ(n)\Theta(n)Θ(n)﻿

7.2 Performance of quicksort

Worst-case partitioning

  • When the partitioning produces subarrays with (n-1) elements and 0 elements
  • T(n)=T(n−1)+Θ(n)T(n) = T(n-1)+\Theta(n)T(n)=T(n−1)+Θ(n)﻿ (arithmetic series) ⇒ T(n)=Θ(n2)T(n)=\Theta(n^2)T(n)=Θ(n2)﻿

Best-case partitioning

  • When the partitioning produces two subarrays of size ⌊n/2⌋\lfloor n/2 \rfloor⌊n/2⌋﻿ and ⌈n/2⌉−1\lceil n/2 \rceil -1⌈n/2⌉−1﻿
  • T(n)=2T(n/2)+Θ(n)T(n)=2T(n/2) + \Theta(n)T(n)=2T(n/2)+Θ(n)﻿ ⇒ T(n)=Θ(nlg⁡n)T(n)=\Theta(n \lg n)T(n)=Θ(nlgn)﻿

Balanced partitioning

  • Any split of constant proportionality yields O(nlg⁡n)O(n \lg n)O(nlgn)﻿: a recursion tree of depth Θ(lg⁡n)\Theta(\lg n)Θ(lgn)﻿ where the cost at each level is O(n)O(n)O(n)﻿

Intuition for the average case

  • When levels alternate between good and bad splits: still O(nlg⁡n)O(n \lg n)O(nlgn)﻿ but with a slightly larger constant hidden by the O-notation.

7.3 Randomized quicksort

  • Random sampling: select a randomly chosen element from the subarray A[p..r] as the pivot
